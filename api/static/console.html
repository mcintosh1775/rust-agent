SecureAgnt Operations Console

M11A baseline dashboard for run health, latency, token usage, payments, and compliance delivery.

No threshold data yet

Console uses existing API RBAC headers (`x-tenant-id`, `x-user-role`). `viewer` is read-restricted and will show per-panel access denials. If trusted proxy auth is enabled, set `Auth Proxy Token` so requests include `x-auth-proxy-token`. Alert acknowledgements send `x-user-id` to `/v1/audit/compliance/siem/deliveries/alerts/ack`. Bootstrap completion requires `agent-id`, `owner` role, and `x-user-id`. Heartbeat materialization actions require `agent-id`; apply mode also requires `x-user-id`.
